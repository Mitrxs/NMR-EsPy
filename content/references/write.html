nmrespy.write¶

Module for the creation of text and PDF files of estimation results.

Requirements for PDF creation¶

To generate PDF result files, it is necessary to have a LaTeX installation set up on your system. For a simple to set up implementation that is supported on all major operating systems, consider TexLive. To ensure that you have a functioning LaTeX installation, open a command prompt/terminal and type:

$ pdflatex --version

The following is a full list of packages that your LaTeX installation will need to successfully compile the .tex files generated by this module:

  • amsmath

$ kpsewhich booktabs.sty

If a pathname appears, the package is installed to that path.

class nmrespy.write.ResultWriter(expinfo, params, errors=None, description=None)[source]¶

Class for writing result files. A simple example of using the class is as follows:

>>> from nmrespy.write import ResultWriter
>>> writer = ResultWriter(
...     expinfo,  # nmrespy.ExpInfo
...     params,  # np.ndarray
...     errors,  # np.ndarray
...     description="Example result file.",  # str
... )
>>> # Write to a text file (defualt if `fmt` isn't given explicitely)
>>> writer.write(path="result", fmt="txt")
Saved file /.../result.txt.
>>> # Write to PDF (suitable LaTeX installation required)
>>> writer.write(path="result", fmt="pdf")
Saved file /.../result.pdf.
You can view and customise the corresponding TeX file at /.../result.tex

Example outputs:

  • result.txt

  • result.tex

  • result.pdf

Parameters

  • expinfo (ExpInfo) – Experiment information.

  • params (Iterable[ndarray]) – Parameters derived from estimation.

  • errors (Optional[Iterable[ndarray]]) – Errors derived from estimation. Should be of the same shape as parameters.

  • description (Optional[str]) – A description to be added to the top of result files.

Return type

None

write(path, fmt='txt', titles=None, experiment_info_sig_figs=5, parameters_sig_figs=5, parameters_sci_lims=(- 2, 3), force_overwrite=False, fprint=True, pdflatex_exe=None)[source]¶

Write result to text file or PDF.

Parameters

  • path (Union[str, Path]) – Path to save the result file to.

  • fmt (str) – Must be one of "txt" or "pdf".

  • titles (Optional[Iterable[str]]) – Titles for each parameter table. If None, give generic titles.

  • experiment_info_sig_figs (Optional[int]) – The number of significant figures to give to numerical experiment inforamtion. If None, the full value will be used.

  • parameters_sig_figs (Optional[int]) – The number of significant figures to give to parameter values. If None, the full value will be used.

  • parameters_sci_lims (Optional[Tuple[int, int]]) – Given a value (-x, y), for ints x and y, any parameter p with a value which satisfies p < 10 ** -x or p >= 10 ** y will be expressed in scientific notation, rather than explicit notation. If None, all values will be expressed explicitely.

  • force_overwrite (bool) – If the file specified already exists, and this is set to False, the user will br prompted to specify that they are happy overwriting the current file.

  • fprint (bool) – Specifies whether or not to print information to the terminal.

  • pdflatex_exe (Optional[Union[str, Path]]) –

    The path to the system’s pdflatex executable.

    Note

    You are unlikely to need to set this manually. It is primarily present to specify the path to pdflatex.exe on Windows when the NMR-EsPy GUI has been loaded from TopSpin.

Return type

None
